## Option 2: Docker PostgreSQL (Simpler Setup)

Create a `docker-compose.yml` in your project root:

```yaml
version: '3.8'
services:
  postgres:
    image: postgres:15-alpine
    container_name: viteElectronAppTemplate-postgres
    environment:
      POSTGRES_DB: viteElectronAppTemplate
      POSTGRES_USER: viteElectronAppTemplate_user
      POSTGRES_PASSWORD: local_dev_password
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./sql/init.sql:/docker-entrypoint-initdb.d/init.sql
```

**Initialize script (`sql/init.sql`):**
```sql
CREATE TABLE patients (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  date_of_birth DATE,
  medical_history JSONB,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE recordings (
  id SERIAL PRIMARY KEY,
  patient_id INTEGER REFERENCES patients(id) ON DELETE CASCADE,
  heart_area VARCHAR(50) NOT NULL CHECK (heart_area IN ('aortic', 'pulmonary', 'tricuspid', 'mitral')),
  file_path VARCHAR(500),
  duration INTEGER, -- in seconds
  analysis_result JSONB,
  status VARCHAR(20) DEFAULT 'processing' CHECK (status IN ('processing', 'completed', 'flagged')),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_recordings_patient_id ON recordings(patient_id);
CREATE INDEX idx_recordings_heart_area ON recordings(heart_area);
CREATE INDEX idx_recordings_status ON recordings(status);
```

## Benefits Over IndexedDB

### Advantages:
1. **SQL Queries**: Complex relational queries and joins
2. **Data Integrity**: Foreign keys, constraints, transactions
3. **Performance**: Better for large datasets and complex operations
4. **Familiarity**: Standard SQL knowledge applies
5. **Backup/Export**: Standard PostgreSQL tools work
6. **Concurrent Access**: Better handling of multiple operations
